エコシステム

エッジゲートウェイとシミュレータが、現場のプロトコルと Building OS をつなぎます。

(gateway_id, point_id) — 共有ポイントリスト契約

Building OS とゲートウェイの間の約束事は、とてもシンプルです。 ポイントリストの正本は Building OS のデジタルツインにあり、ゲートウェイは 現場ネイティブのアドレス(BACnet のオブジェクト番号や OPC UA のノード ID など)を 自分の側で point_id に解決します。ワイヤ上を流れるのは gateway_id + point_id + value + timestamp だけ — プロトコルの詳細は 現場側に閉じ込められます。

ゲートウェイは GET /gateways/{id}/pointlist でポイントリストを同期します (ETag による差分検知つきなので、変更がないときの問い合わせはほぼタダです)。接続は mTLS によるマシン認証で保護されます。

関連プロダクト

  • nexus-gateway

    参照実装のエッジ統合ゲートウェイ(Go)。BACnet / OPC UA / MQTT のプロトコルコネクタを ホストし、ネイティブアドレスを point_id に正規化します。SQLite の store-and-forward バッファで通信断にも耐え、上りは GatewayIngress、下りは GatewayEgress に接続します。

    gutp-bim/nexus-gateway →

  • bacnet-sim-gateway

    SBCO ポイントリストから、標準準拠の仮想 BACnet コントローラ(B-BC)を生成して BACnet/IP で公開するシミュレータ(Python 3.12)。物理設備がなくても、BACnet 経由の E2E 検証ができます。

    takashikasuya/bacnet-sim-gateway →

  • opcua-sim-gateway

    SBCO ポイントリストから仮想 OPC UA サーバを生成し、opc.tcp で公開するシミュレータ (Python 3.12)。工場・プラント系でよく使われる OPC UA 経由の検証に使えます。

    takashikasuya/opcua-sim-gateway →

物理設備ゼロで、現場を再現する

Building OS 本体・nexus-gateway・2 つのシミュレータを組み合わせると、物理設備なしで 「センサーの値が上がってきて、制御が現場プロトコルまで届く」一連の流れを再現できます。 研究・検証・デモに便利です。
