ビル運用のデータ基盤を、
自分たちの手に。

ベンダーロックインなしで、設備データの収集・蓄積・可視化・制御をワンストップに。 Building OS は、スマートビルの「データの土台」になるオープンソースプラットフォームです。

スマートビルの現場が抱える課題

  • データのサイロ化

    空調は空調の、電力は電力のシステムに閉じてデータが分断され、横断的な分析や活用が難しい。

    → すべての設備データを共通のデータモデルと単一の API に集約します。

  • クラウド・ベンダー依存

    特定ベンダーのマネージドサービスに深く依存すると、コストと事業継続性のリスク、 そしてデータ主権の問題がつきまとう。

    → 全部オープンソース。オンプレでも任意のクラウドでも自前で運用できます。

  • プロトコルの差異

    BACnet / OPC UA / MQTT など、現場ごと・設備ごとに接続方式が異なり、つなぎ込みのたびに 個別開発が発生する。

    → ゲートウェイ層がプロトコル差異を吸収し、共通の契約でつながります。

誰のためのプラットフォームか

  • ビルオーナー・運用者

    設備データを自社の資産として蓄積。既存のビル管理システム(BAS)の横に置ける、 独立したデータ基盤として使えます。

  • 実践者(SIer・アプリ開発者)

    REST + gRPC の API、Swagger 定義、生成済みクライアントで、ビルデータを使った アプリケーションをすぐに作り始められます。

  • 研究者・教育

    すべてオープンソースで、すべてローカルで再現可能。デジタルツインや 建物オントロジー研究の実験台としても使えます。

主要機能

建物の構造をそのままデータモデルに

建物→フロア→空間→機器→ポイントの階層をデジタルツイン(OxiGraph / SPARQL)として管理。 管理画面から Turtle ファイルをアップロードして、ツインの構築・更新ができます。

最新値も 1 年前も、同じ API で

直近の値は Hot(NATS KV)から、過去の履歴や集計は Parquet レイクから — データの置き場所は API が自動選択します。利用者はポイントを選んで期間を指定するだけです。

制御は、確実に・追跡可能に

ポイント単位の制御はゲートウェイ別にルーティングされ、結果は非同期で返ります。 オフラインのゲートウェイには待たされずに即時エラー(503)。すべての制御操作は 監査証跡としてデータベースに記録されます。

Keycloak によるエンタープライズ認証

OIDC / JWT によるユーザー認証と、ロール・グループに基づくリソース単位の認可。 「このユーザーはこの建物のこのフロアまで」という粒度でアクセスを制御できます。

プラットフォームの今を一目で

API・ワーカー・メッセージバスなど各コンポーネントの稼働状態を、ダッシュボードで ひと目で確認。外部の監視基盤がなくても、基本の運用は built-in の画面で回せます。

運用の解像度を上げる

より深く見たいときは、OpenTelemetry → Prometheus / Grafana / Loki / Tempo の 可観測性スタックをオプションで有効化。メトリクス・ログ・トレースを横断して追えます。
