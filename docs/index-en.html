Building OS — OSS Edition collects telemetry from building equipment (HVAC, power meters, environmental sensors), stores it in a Parquet lake, and serves it through REST + gRPC APIs and a Next.js dashboard — with point-level control and a full audit trail.

What it is

A self-hostable IoT platform that turns building-equipment data into an asset you own — no managed-cloud lock-in. The entire stack is open source: NATS, MinIO, OxiGraph, Keycloak, and PostgreSQL, orchestrated by .NET workers and served to a Next.js web client.

  • Digital twin at the core — the building → floor → space → device → point hierarchy lives in OxiGraph (RDF / SPARQL) using the SBCO ontology, with a documented mapping to Brick, REC, IFC, and DTDL.
  • Tiered telemetry — latest values in NATS KV (hot), history and aggregates in a Parquet lake on MinIO (warm / cold). A single GET /telemetries/query endpoint picks the right tier automatically.
  • Point-level control — per-gateway routing, immediate 503 fast-fail for offline gateways, and a complete audit trail in PostgreSQL.
  • Enterprise auth & observability — Keycloak (OIDC / JWT) for users, mTLS machine auth for gateways, and an opt-in OpenTelemetry → Prometheus / Grafana / Loki / Tempo stack.

Why it exists

Building OS — OSS Edition is a derivative of research output from the UTokyo Green ICT Project (GUTP). The original research platform depended on managed cloud services; this edition re-architects it onto a fully open-source stack so that practitioners and researchers can run it anywhere — on premises or in any cloud — under their own control.

Quick start

git clone https://github.com/gutp-bim/gutp-building-os-ri
cd gutp-building-os-ri
docker compose -f docker-compose.oss.yaml up -d
docker compose -f docker-compose.oss.yaml --profile webclient up -d
# Web:     http://localhost:3000
# API:     http://localhost:5000
# Swagger: http://localhost:5000/swagger

# Read the latest value of a point
curl 'http://localhost:5000/telemetries/query?pointId=demo-pt-001&latest=true'

# Send a (simulated, by default) control command
curl -X POST 'http://localhost:5000/points/demo-pt-001/control' \
  -H 'Content-Type: application/json' -d '{"value": 1}'

For the full walkthrough (twin import, telemetry ingestion, troubleshooting), see Getting Started (Japanese) or the README English Summary.

Ecosystem

Building OS talks to the field through a simple contract: the digital twin is the source of truth for the shared point list, and only gateway_id + point_id + value + timestamp travel on the wire. The reference edge gateway, nexus-gateway (Go), hosts BACnet / OPC UA / MQTT connectors and normalizes native addresses to point IDs. Two simulators — bacnet-sim-gateway and opcua-sim-gateway — let you reproduce a full field-to-dashboard pipeline with zero physical equipment.
