Building OS OSS Edition
ホーム プロダクト クイックスタート アーキテクチャ エコシステム EN GitHub

15 分で、テレメトリが流れ出す。

必要なのは Docker Desktop だけ。収集 → 蓄積 → 読み出し → 制御までの一連の流れを、 ローカルマシンで体験できます。

⚠️ 本ソフトウェアは現状有姿(AS IS)・無保証です。なお OSS 版の既定は ENABLE_SIM_CONTROL=true(シミュレート制御)のため、このクイックスタートで 実際の設備が動くことはありません。実機に接続する場合は必ず 免責事項を確認し、 利用者自身の責任で十分に検証してください。

0. 前提条件

ツール必須?用途
Docker Desktop必須全サービスを Docker Compose で起動
Python 3推奨サンプルデバイス(MQTT シミュレータ)の実行
.NET SDK 8.0.x / Node.js 22+任意API / Web をホストで直接動かして開発する場合のみ

1. OSS スタックを起動する

リポジトリを clone して、Docker Compose で一式を立ち上げます。

git clone https://github.com/gutp-bim/gutp-building-os-ri
cd gutp-building-os-ri
docker compose -f docker-compose.oss.yaml up -d

これで次のサービスが起動します(既定構成)。

サービス役割ローカルポート
NATS JetStreamコアメッセージバス4222 / 8222
PostgreSQL 16ユーザー・グループ・監査5433

2. ダッシュボードと API を開く

Web クライアントは webclient プロファイルで追加起動します。

docker compose -f docker-compose.oss.yaml --profile webclient up -d
# Web:     http://localhost:3000
# API:     http://localhost:5000
# Swagger: http://localhost:5000/swagger

サンプルのデジタルツイン(デモビル)は起動時に自動投入されます。 http://localhost:3000/resources を開くと、建物→フロア→空間→機器→ポイントの 階層をたどれます。

3. テレメトリを流す

いちばん簡単なのは MQTT 経由です。Mosquitto プロファイルを有効化して起動します。

MQTT_HOST=building-os.mosquitto docker compose -f docker-compose.oss.yaml --profile mqtt up -d

付属のデバイスシミュレータで 1 件 publish してみます。

MQTT_HOST=localhost MQTT_PORT=11883 MQTT_USERNAME=devices MQTT_PASSWORD=buildingos-devices \
  TELEMETRY_INTERVAL=1 DEVICE_ID=device-001 TENANT_ID=default POINT_ID=PT001 \
  python Tools/development-edge-device/mqtt_edge_device.py

※ ゲートウェイからの本来の取り込み経路は gRPC(GatewayIngress)です。gRPC 経由・持続負荷・ Parquet レイクへの反映確認などの詳細手順は getting-started.md を参照してください。

4. API で読み出す

統一エンドポイント GET /telemetries/query が、最新値・履歴・集計の置き場所を自動選択します。

# 最新値(Hot KV)
curl 'http://localhost:5000/telemetries/query?pointId=demo-pt-001&latest=true'

# 期間レンジ(tier 自動選択: warm / cold / 集計)
curl 'http://localhost:5000/telemetries/query?pointId=demo-pt-001&start=2026-06-15T00:00:00Z&end=2026-06-16T00:00:00Z&granularity=Hour'

※ pointId はデジタルツインに登録済みのポイント ID を指定します(未登録は 404)。

5. ポイントを制御する

curl -X POST 'http://localhost:5000/points/demo-pt-001/control' \
  -H 'Content-Type: application/json' -d '{"value": 1}'

202 Accepted と controlId が返り、実行結果は非同期で通知されます。 既定はシミュレート制御なので、安全に流れを確認できます。オフラインのゲートウェイ宛ての制御は 待たされずに即時 503 が返ります。
